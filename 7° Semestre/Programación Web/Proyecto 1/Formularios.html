Formularios

Los formularios en HTML son una parte esencial de la interacción entre un usuario y un sitio web o aplicación. Permiten a los usuarios introducir datos, que generalmente se envían a un servidor web para su procesamiento y almacenamiento.

Elementos básicos de un formulario HTML

  • <form>: Es el contenedor de los controles del formulario.
  • <input>: Crea campos de texto de una o varias líneas, cajas desplegables, botones, casillas de verificación o botones de opción.
  • <label>: Proporciona una descripción textual de los controles de formulario.

    • Atributos

  • action="": Entre comillas se indica la acción a realizar al enviar el formulario. En general se indicará el nombre de un fichero alojado en el servidor, el cual se encargará de procesar la información. Aunque también se le puede indicar una dirección de correo para que envíe directamente todo el contenido, de la forma: mailto:direccion_de_correo.
  • method="" (post o get): Indica el método de transferencia de las variables. El método "post" envía los datos de forma no visible, mientras que el método "get" los adjunta a la URL a la que se redirige.
  • enctype= "": Especifica el tipo de codificación de la información enviada. Con method="get" no se realiza codificación, solo se cambian caracteres especiales como el espacio, por lo que no es necesario indicar enctype. Cuando el valor del atributo "method" es "post", podemos utilizar los siguientes valores:
    • application/x-www-form-urlencoded: Es el valor predeterminado. Codifica todos los caracteres antes de enviarlos.
    • multipart/form-data: Es requerido al enviar archivos mediante un formulario. No codifica la información.
    • text/plain: No codifica la información, solo cambia los espacios por el símbolo "+".

Diseño de formularios

Antes de comenzar a escribir código, es importante pensar en tu formulario. Diseñar una maqueta rápida te ayudará a definir el conjunto de datos adecuado que deseas pedirle al usuario que introduzca. Recuerda que cuanto más grande es tu formulario, más te arriesgas a frustrar a las personas y perder usuarios. Debe ser simple y conciso: solicita solo los datos que necesitas.

Ejemplo

<form action="ruta_del_procesador.php" method="post"> <label for="nombre">Nombre:</label><br> <input type="text" id="nombre" name="nombre"> <input type="submit" value="Enviar"> </form>

<form action="" method=""> <fieldset> <legend>Opcion 1</legend> <input type="radio" name="radio" id="radio"> <label for="radio">Clic Aquí</label> <legend>Opcion 2</legend> <input type="radio" name="radio" id="radio"> <label for="radio">Clic Aquí</label> </fieldset> </form>

Este formulario contiene un campo de texto para el nombre del usuario y un botón de envío. Cuando el usuario hace clic en el botón de envío, los datos del formulario se envían al “agente procesador” especificado en el atributo action del elemento <form>, utilizando el método HTTP especificado en el atributo method.
